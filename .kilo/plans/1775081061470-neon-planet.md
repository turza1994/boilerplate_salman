# Social Media Feed Application - Implementation Plan

## Overview
Convert the fullstack boilerplate into a social media application with authentication, posts, comments, likes, and a feed. The existing boilerplate provides auth (JWT), Express backend, React frontend, Drizzle ORM, and PostgreSQL.

## Current State
- Auth system exists (email/password only, no firstName/lastName)
- DB schema already defined with users, posts, comments, postLikes, commentLikes tables
- Frontend has Login/Signup forms, Dashboard page, AuthContext, apiClient
- Backend has layered architecture: Routes → Controllers → Services → Repositories

## Phase 3: Backend - Post API

### 3.1 Post Validation Schemas (`packages/backend/src/schemas/post.ts`)
- `createPostSchema`: content (string, min 1), visibility ('public' | 'private')
- `updatePostSchema`: content (optional), visibility (optional)

### 3.2 Post Repository (`packages/backend/src/repositories/postRepository.ts`)
- `createPost(userId, content, imageUrl, visibility)` - INSERT returning
- `getPostById(id)` - SELECT with author info
- `getFeedPosts(userId, cursor, limit)` - SELECT paginated, newest first
  - Public posts from all users + private posts from current user
  - Include author info, like count, comment count, whether current user liked
- `updatePost(id, userId, data)` - UPDATE returning (owner only)
- `deletePost(id, userId)` - DELETE (owner only)
- `getPostWithDetails(id, currentUserId)` - Full post with likes, comments

### 3.3 Post Service (`packages/backend/src/services/postService.ts`)
- Business logic for CRUD operations
- Authorization checks (owner-only edit/delete)
- Visibility filtering

### 3.4 Post Controller (`packages/backend/src/controllers/postController.ts`)
- CRUD endpoints following existing patterns

### 3.5 Post Routes (`packages/backend/src/routes/postRoutes.ts`)
- `POST /api/posts` - Create (auth required, CSRF)
- `GET /api/posts` - Feed (auth required, query: cursor, limit)
- `GET /api/posts/:id` - Single post (auth required)
- `PUT /api/posts/:id` - Update (auth required, CSRF)
- `DELETE /api/posts/:id` - Delete (auth required, CSRF)

## Phase 4: Backend - Comment API

### 4.1 Comment Validation Schemas (`packages/backend/src/schemas/comment.ts`)

### 4.2 Comment Repository (`packages/backend/src/repositories/commentRepository.ts`)

### 4.3 Comment Service (`packages/backend/src/services/commentService.ts`)

### 4.4 Comment Controller (`packages/backend/src/controllers/commentController.ts`)

### 4.5 Comment Routes (`packages/backend/src/routes/commentRoutes.ts`)

## Phase 8: Frontend - Types & API Client

### 8.1 Post Types (`packages/frontend/src/types/post.ts`)
```typescript
interface Post {
  id: number; userId: number; content: string; imageUrl: string | null;
  visibility: 'public' | 'private'; createdAt: string; updatedAt: string;
  author: { id: number; firstName: string; lastName: string };
  likeCount: number; commentCount: number; isLiked: boolean;
}
interface Comment {
  id: number; postId: number; userId: number; content: string;
  parentId: number | null; createdAt: string;
  author: { id: number; firstName: string; lastName: string };
  likeCount: number; isLiked: boolean; replies: Comment[];
}
interface FeedResponse { posts: Post[]; nextCursor: number | null; }
```

### 8.2 Update API Client (`packages/frontend/src/lib/apiClient.ts`)
- Add post methods: createPost, getFeed, getPost, updatePost, deletePost
- Add comment methods: createComment, getComments, deleteComment, replyToComment
- Add like methods: togglePostLike, toggleCommentLike, getPostLikers, getCommentLikers
- Add upload method: uploadImage
